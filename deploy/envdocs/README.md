# deploy/envdocs

Manifests for the environment documentation site at
`https://envdocs.usgov.coderdemo.io`: a static MkDocs Material site served by
nginx in-cluster, gated by oauth2-proxy against Keycloak realm `coder` (any
authenticated realm user, no group restriction).

These manifests are applied by `scripts/setup-envdocs.py`, which also creates the
`envdocs` Keycloak OIDC client, publishes its secret to AWS Secrets Manager,
mirrors the required images into ECR, generates the site ConfigMap from
`docs/envdocs/`, and adds the Route53 alias. Do not apply by hand without that
context; the `envdocs-site` ConfigMap is generated by the script and is not
committed.

## Files

| File | Purpose |
|---|---|
| `namespace.yaml` | Namespace `envdocs` (Istio injection disabled; reached via ingress-nginx) |
| `externalsecret.yaml` | ESO ExternalSecret `envdocs-oauth` from ASM `usgov-coderdemo/envdocs/oauth` (keys `client-secret`, `cookie-secret`) |
| `deployment.yaml` | `envdocs` Deployment (mkdocs-material init build + nginx) and its Service |
| `oauth2-proxy.yaml` | `oauth2-proxy` Deployment + Service (OIDC auth gate) |
| `ingress.yaml` | Two Ingresses on `envdocs.usgov.coderdemo.io`: `/oauth2/*` to oauth2-proxy, `/` to nginx with external-auth annotations |

## Generated, not committed

- `ConfigMap/envdocs-site` (ns `envdocs`): generated by `scripts/setup-envdocs.py`
  from `docs/envdocs/mkdocs.yml` and `docs/envdocs/docs/*.md`. The single source
  of truth is `docs/envdocs/`.
- `Secret/envdocs-oauth` (ns `envdocs`): owned by ESO from ASM.

## Images (mirrored into ECR)

GovCloud has no ECR pull-through cache, so `scripts/setup-envdocs.py` mirrors:

| Upstream | ECR path |
|---|---|
| `docker.io/library/nginx:1.27-alpine` | `docker-hub/library/nginx:1.27-alpine` |
| `docker.io/squidfunk/mkdocs-material:9.7.6` | `docker-hub/squidfunk/mkdocs-material:9.7.6` |
| `quay.io/oauth2-proxy/oauth2-proxy:v7.7.1` | `quay/oauth2-proxy/oauth2-proxy:v7.7.1` |

## Routing and DNS

The Route53 wildcard `*.usgov.coderdemo.io` aliases to the Istio gateway NLB.
This site instead uses an explicit `envdocs.usgov.coderdemo.io` alias to the
ingress-nginx NLB (a more specific record wins), so envdocs traffic flows through
ingress-nginx where the external-auth annotations live. Both NLBs carry the same
`*.usgov.coderdemo.io` ACM certificate.

## Apply

```sh
. ~/.config/usgov-coderdemo/env
export KUBECONFIG=/home/coder/demoenv-workspace/usgov-coderdemo/kubeconfig
python3 scripts/setup-envdocs.py
```

The script is idempotent and safe to re-run. See `scripts/setup-envdocs.py`
`--help` for phased flags (`--plan`, `--skip-mirror`, `--skip-dns`).
